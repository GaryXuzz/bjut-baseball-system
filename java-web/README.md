# BJUT Baseball Java Web Version

This directory contains the Java + MySQL + Web UI implementation for COMP2004J Phase 2.

It uses:

- Java built-in `com.sun.net.httpserver.HttpServer`
- JDBC
- MySQL
- plain HTML generated by Java

No Maven, Gradle, Spring, or Tomcat is required.

Java 17 or newer is recommended.

## 1. Prepare MySQL

Create the database:

```sql
CREATE DATABASE bjut_baseball
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;
```

Import the schema from the repository root:

```bash
mysql -u root -p bjut_baseball < sql/phase2_mysql_schema.sql
```

Optional small demo data. The seed file is safe to run more than once for the included demo records:

```bash
mysql -u root -p bjut_baseball < java-web/demo_seed.sql
```

Original project data from the earlier Flask/SQLite prototype:

```bash
mysql -u root -p bjut_baseball < sql/original_project_data.sql
```

Use this original data import for the Phase 2 demonstration. It loads the real project player list, position relationships, batting/pitching profiles, and game records while keeping the administrator account from the schema.

The Java demo accepts the configured `auth.demo.username` and `auth.demo.password` in `config.properties`, so the default login is `admin / admin123`. This is intentionally simple for course demonstration. The earlier Flask prototype can keep its hashed password flow separately.

## 2. Add MySQL Connector/J

Download MySQL Connector/J and place the jar file in:

```text
java-web/lib/mysql-connector-j.jar
```

The Java source uses only standard JDBC interfaces, so it can compile without the jar. The jar is required at runtime when connecting to MySQL.

## 3. Configure Database Connection

Edit `java-web/config.properties`:

```properties
server.port=8080
db.url=jdbc:mysql://127.0.0.1:3306/bjut_baseball?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
db.user=root
db.password=your_mysql_password
auth.demo.username=admin
auth.demo.password=admin123
```

## 4. Run

macOS / Linux:

```bash
cd java-web
chmod +x run.sh
./run.sh
```

Windows:

```bat
cd java-web
run.bat
```

Open:

```text
http://localhost:8080/
```

## Implemented Pages

- `/` dashboard
- `/login` administrator login
- `/players` player list
- `/players/new` add player
- `/players/edit?id=...` edit player
- `/records/new` add game record
- `/stats` batting and pitching leaderboards with lightweight CSS bar charts
- `/matchups` matchup search by opponent and player
- `/records/delete` delete one game record as administrator
- `/export/players.csv` export player summary data
- `/export/game-records.csv` export game records
- `/schema` database table and constraint summary

The UI includes responsive navigation and horizontally scrollable tables for smaller screens.
